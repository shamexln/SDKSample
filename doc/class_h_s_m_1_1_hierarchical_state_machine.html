Detailed Description

Base class for hierarchical state machines.

Hierarchical state machines allow states to be nested within other states. This is useful for defining common behaviour and reduces code duplication. For example: A state machine which controlls a ui dialog would have two states on the top level: hidden and visible. All the functionality that the dialog provides would be realized as substates of visible. And no matter what state that state machine is in, when the event hide is received it transits to hidden. This reaction has only to be defined for the superstate visible and every substate inherits this behaviour.

const std::string &  name = "aStateMachine" 
)
explicit

Constructs statemachine which is registered at the given container.

After construction this statemachine is in the state named Top. Every received event is ignored in that state. After the init function of the container is called this statemachine transits in its initial state specified by HSM::HierarchicalStateMachine::initial() and is ready to work. The name is used for debug purposes only.
