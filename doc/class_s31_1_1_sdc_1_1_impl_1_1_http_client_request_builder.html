◆ allowRetry()

HttpClientRequestBuilder&& S31::Sdc::Impl::HttpClientRequestBuilder::allowRetry ( bool  allowed) &&

Specify whether the http client may retry a request.

For example, a remote peer may close a keep-alive connection when the http client attempts to send a request. To recover from this asynchronous close event, the client will by default attempt to re-open the connection and retry the message send. However, the client may not be able to determine if the request was received by the remote peer or not. If a request must not be retried due to side effects of being received by the peer, then this option must be set to false.

Note
Retry is allowed by default.

◆ createBodyFunction()

HttpClientRequestBuilder&& S31::Sdc::Impl::HttpClientRequestBuilder::createBodyFunction ( CreateRequestBody  fn,
HttpContentType  contentType 
) &&

Set a create body function and content-type.

A CreateRequestBody function provides possibly dynamically generated body content for the request. Only one option for providing a request body may be provided, therefore a create body function is mutually exclusive with providing a bodyString(). The content-type must also be specified.

Exceptions
S31::InternalConsistencyFailureif the provided function is empty (i.e. not callable).
Note
If a request is retried (e.g. due to a closed connection) then the provided function may be called twice (once for each attempt). The retried connection may provide different context information (e.g. local IP address). See allowRetry() to disable retry attempts. If a precondition check of the server fails when attempting a request, then the provided function will not be called since the request is not made. See verifyPreconditionsFunction().

◆ verifyPreconditionsFunction()

HttpClientRequestBuilder&& S31::Sdc::Impl::HttpClientRequestBuilder::verifyPreconditionsFunction ( VerifyRequestPreconditions  fn) &&

Optionally set a function to provide additional precondition checks.

Setting this function is optional for a request. Example usage is to provide a function which performs additional EKU role checks on the remote peer credentials before reusing an existing keep-alive connection.

Note
The provided function applies to only one client request. If the precondition check fails the connection will remain open for keep-alive reuse by subsequent requests.
