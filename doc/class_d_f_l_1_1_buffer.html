Detailed Description

template<std::size_t bytes>
class DFL::Buffer< bytes >

An immutable, owning STL compliant container for holding bytes.

Being an owning container means that all data given to this buffer in the ctor is copied into the internal storage of this class. Immutable means that once a buffer is constructed you cannot change any values it holds. This class is espacially useful when dealing with c-array APIs. It provides a smooth transition between c++-containers and c-arrays. Thus this container can be constructed in various different ways. From c-array, stl-containers, iterator-ranges, ...

const unsigned char arrAsciiNums[] =
{0x30, 0x31, 0x32, 0x33, 0x34, 0x35, 0x36, 0x37, 0x38, 0x39};
enum
{
asciiNumsSize = (sizeof(arrAsciiNums) / sizeof(arrAsciiNums[0]))
};
const std::string strAsciiNums(arrAsciiNums, arrAsciiNums + asciiNumsSize);
using MyBuffer = ::DFL::Buffer<asciiNumsSize>;

A size missmatch is either detected at compile time ...

// source buffer is too big
// const MyBuffer::Data bufData =
// {
// 0x30, 0x31, 0x32, 0x33, 0x34, 0x35, 0x36, 0x37, 0x38, 0x39, 0x3a
// };
// size of value types does not match
// const unsigned short bufData[] =
// {
// 0x30, 0x31, 0x32, 0x33, 0x34, 0x35, 0x36, 0x37, 0x38, 0x39
// };
// const MyBuffer buf2(bufData, asciiNumsSize);

... or at runtime

{
using MySmallerBuffer = ::DFL::Buffer<asciiNumsSize - 1>;
// NOLINTNEXTLINE(cppcoreguidelines-avoid-goto)
ASSERT_THROW(MySmallerBuffer(arrAsciiNums, asciiNumsSize),
std::runtime_error);
const std::string tooLongStr = strAsciiNums;
// NOLINTNEXTLINE(cppcoreguidelines-avoid-goto)
ASSERT_THROW((MySmallerBuffer(tooLongStr)), std::runtime_error);
// NOLINTNEXTLINE(cppcoreguidelines-avoid-goto)
ASSERT_THROW(MySmallerBuffer(tooLongStr.begin(), tooLongStr.end()),

}

Another compile time check ensures that the type held by the source container is one byte. During construction the data from the source container is copied into the internal storage of the buffer class. Afterwards the data can no longer be modified. Iterators are provided for retrieving the values. And for iterfacing with c-array a pointer to the held data and a size accessor are provided.
