Detailed Description

Provides access to credentials of a communication peer.

In this class all credential handling is aggregated.

Member Function Documentation

◆ allRoles()

DFL::Oids S31::Nomenclature::Credentials::allRoles ( ) const
noexcept

Returns all roles associated with these Credentials.

These roles come from the peer certificate. They are not filtered by whitelist or role authorization list - i.e. if a role is listed here then this does not imply (in some cases) that the peer is trusted or authorized for the role functionlity (e.g. remote control releated roles).

◆ rolesDebugString()

std::string S31::Nomenclature::Credentials::rolesDebugString ( ) const

Debug string representation containing roles from certificate.

Should be used for human readable messages (e.g., logs) only. Not all roles from the certificate are allowed to be used. This might depend on other certificate information like which root signed the certificate chain etc.

◆ sanitizedCommonName()

std::string S31::Nomenclature::Credentials::sanitizedCommonName ( ) const

Special characters (e.g., non-printable ASCII) are hex-encoded.

In a common name of a certificate the whole range of UTF-8 code points is allowed. To make sure that any postprocessing software (e.g., of log files) has no issue "special" characters are replaced. The function uses encoding according RFC 4514 as this exists in DFL.
