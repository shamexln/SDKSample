◆ actualPriority()

boost::optional< AlertConditionPriority > DFL::Mdib::AbstractAlertConditionState::actualPriority ( ) const
inline

Current priority of the alert condition that has been modified by an escalation or deescalation process.

If the actual priority is present in an alert condition state, the priority of the related alert condition descriptor does not reflect the current severity of the potential or actual hazard that exists if the alert condition is present. If there is no actual priority given, the value is boost::none.

◆ presence()

AlertConditionPresence DFL::Mdib::AbstractAlertConditionState::presence ( ) const
inline

Alert condition presence.

Note on changing the presence of an alert condition

AlertSystem's state has 2 fields that represent the presence of it's alert conditions. A change in condition's presence has to be reflected in this AlertSystemState properties.

See: AlertSystemState::presentPhysiologicalAlarmConditions(), AlertSystemState::presentTechnicalAlarmConditions()

There is helper functionality that can update these lists before writing a changeset into a Mdib.

See: AlertHelper::setAlertSystemConditionPresenceLists()
