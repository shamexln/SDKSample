Detailed Description

A task bound to some executor(-provider).

Generally the task has a delay after which it's started but this delay can also be 0ms, which means the task is to be executed immediatly. Of course this depends on the executor how 'immediatly' this is. If the Task is destroyed before the timer finishes then the TaskFunction may not be executed at all.
