Detailed Description

Information for an IPv4 network adapter.

This is a container for all kind of network adapter information.

EXPECT_EQ(m_adapterA.index(), 1);
EXPECT_EQ(m_adapterA.name(), "adapter_a");
EXPECT_EQ(m_adapterA.isLoopback(), false);
EXPECT_EQ(m_adapterA.isMulticast(), true);
EXPECT_EQ(m_adapterA.isConnected(), true);
EXPECT_EQ(m_adapterA.addressesV4(),
(AddressesV4{AddressV4{"1.2.3.4"}, AddressV4{"11.22.33.44"}}));
EXPECT_EQ(m_adapterA.macAddress(), (MacAddress{{1, 2, 3, 44, 55, 66}}));

◆ addressesV4()

Ip::AddressesV4 DFL::Net::Adapter::Info::addressesV4 ( ) const

Provides IPv4 addresses.

Typically a disconnected adapter has no IPv4 address. As soon as it detects a media connection an IPv4 address is assigned either by DHCP or APIPA (Automatic Private IP Addressing).

In "server" use cases IPv4 addresses might be managed/configured manually. In these scenarios it is possible to have multiple IPv4 addresses per adapter. A typical scenario for multiple IPv4 addresses are HTTPS server hosting multiple different domains. For such a domain an individual IPv4 address is used.

◆ index()

Index DFL::Net::Adapter::Info::index ( ) const

Provides the unique and constant index.

Typical use cases for an interface index is to specify an outgoing network adapter for multicast messages. In that case it is not possible to figure out the adapter from the destination address.

◆ isConnected()

bool DFL::Net::Adapter::Info::isConnected ( ) const

Provides the media connection state.

A typical ethernet adapter might have a cable plugged in or removed. This is the "connected" status of such an adapter.

  • false: cable disconnected or interface disabled
  • true: cable connected and interface is ready to use

◆ isMulticast()

bool DFL::Net::Adapter::Info::isMulticast ( ) const

Provides information about multicast support.

Not all network adapter support multicast. Some are tunnel adapter for example and do not allow to transmit multicast data.

◆ name()

const std::string& DFL::Net::Adapter::Info::name ( ) const

Retrieves the unique name of this adapter.

Depending on the implementation the name might change during runtime. To identify an already retrieved adapter use the index, because it does not change.

Note
