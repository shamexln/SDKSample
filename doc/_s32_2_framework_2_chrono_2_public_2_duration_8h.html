Detailed Description

Defines different duration types with different periods and cast function (duration_cast) to convert a duration with a higher resolution to one with a lower resolution.

Durations represent a timespan like 2 hours or 12 seconds and 100 milliseconds. All durations are arithmetic types which behave much like an int.

using namespace ::DFL::Chrono;
using namespace ::DFL::Chrono::Literals;
auto s1{Hours(1) + Minutes(2) + Seconds(4)};
const auto s2{1_h + 2_min + 4_s};
ASSERT_EQ(s1, 3724_s);
ASSERT_EQ(s1, s2);
ASSERT_EQ(s1 + s2, 2 * s1);
ASSERT_EQ(s1 - s2, 0_s);
ASSERT_EQ(s1 / 2, 1862_s);
++s1;
--s1;
ASSERT_EQ(s1, s2);

    Defining duration with custom periods is simple too. A duration with a ratio of <1, 1> corresponds to seconds. Thus if an application is called periodically every 250 ms. Then one can simple define a tick duration,
    using Tick = duration< int, ratio<1, 4> >;
    thus making it unecessary to compute a tick counter from a given duration.
    {
    const Tick t = Seconds(2);
    ASSERT_EQ(8, t.count());
    const Seconds s(duration_cast<Seconds>(t));
    ASSERT_EQ(Seconds(2), s);
    }
    {
    const Tick t = duration_cast<Tick>(Milliseconds(750));
    ASSERT_EQ(3, t.count());
    const Milliseconds ms(t);
    ASSERT_EQ(Milliseconds(750), ms);
    }
