Detailed Description

Is used by an SDC ALERT PROVIDER in order to allow an SDC ALERT CONSUMER to use this list for visualizing its own parameter boxes. If this extension is not given it has to be assumed that the default flashing behavior can be used by the SDC ALERT CONSUMER (i.e. flashing when a pm:AlertSignalState/@Presence = true, not flashing when pm:AlertSignalState/@Presence = false, during Audio Pause only flash for "new" signals).

This list is also required, if accurate flashing states for SDC ALERT CONSUMERs that join while Audio Pause situations are already active are needed.

Example usage in code:

// Device side usage: create the AlertSignalFlashing instance and add it
// after conversion with the free function
// DFL::Mdib::Ext::ParticipantModelExtensions::toExtension to any extendable
// MDIB class
const AlertSignalFlashing deviceAsf{
{DFL::Mdib::Handle{"handle1"}, DFL::Mdib::Handle{"handle2"}}};
DFL::Mdib::AlertSystemState const ass{
DFL::Mdib::AlertActivation::On,
boost::none,
boost::none,
{},
{},
// just add the extension in the ctor an MDIB class
{DFL::Mdib::Ext::ParticipantModelExtensions::toExtension(deviceAsf)}};
// Client side usage: use the free function
// DFL::Mdib::Ext::ParticipantModelExtensions::get<AlertSignalFlashing>() to
// retrieve the AlertSignalFlashing from any state or other extendable
// element from the MDIB.
if (const auto clientAsf =
ParticipantModelExtensions::get<AlertSignalFlashing>(ass))
{
EXPECT_THAT(clientAsf->flashingSignals(), SizeIs(2));
EXPECT_THAT(clientAsf->flashingSignals(),
Eq(deviceAsf.flashingSignals()));
}
