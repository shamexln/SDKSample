Detailed Description

Class for implementing the safe bool idiom.

The naive approach on converting a handcrafted type to a bool would be the usage of a conversion operator:

class ToBool
{
public:
ToBool(int value) : m_value(value) {}
int m_value;
operator bool() const
{
return m_value > 0;
}
};

Now you can use instances of this type in an if expression:

ToBool const isTrue(1);
ToBool const isFalse(-1);
if(isTrue) { assert(true); }
else { assert(false); }
if(isFalse) { assert(false); }
else { assert(true); }

But there are a lot of more expressions available:

isTrue < isFalse;
isTrue + isFalse;

All kinds of relational- and arithmetic operators work with this type. The reason is that bool is implicitly convertible to int and since ToBool is implicit convertible to bool the conversion to int works aswell. A version of ToBool which turns these experssions into a compiletime errors looks like this:

class ToBool
{
public:
ToBool(int value) : m_value(value) {}
int m_value;
operator SafeBool::Type() const
{
return SafeBool::isTrue(m_value > 0);
}
};

Since this idiom is very useful it has direct support from the language since c++11. The feature is called explicit conversion operator and looks like this:

class ToBool
{
public:
ToBool(int value) : m_value(value) {}
int m_value;
explicit operator bool() const
{
return m_value > 0;
}
};

To take advantage of this two macros are provided. On pre c++11 compiler the SafeBool class is used and on c++11 compilers the explicit conversion operator is used:

// ToBool.h
class ToBool
{
DFL_SAFEBOOL_DECL();
};
// ToBool.cpp
DFL_SAFEBOOL_IMPL(ToBool, (m_value > 0))
Warning
If one has to write code written with and without c++11 support take into account that explicit conversion operators only work in direct initialization context whilst the safe bool idiom also works in copy initialization contexts.
ToBool const isTrue(1);
bool a(isTrue); // direct initialization
bool b = isTrue; // copy initialization
