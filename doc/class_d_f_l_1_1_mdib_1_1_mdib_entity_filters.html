Filter function chaining over parents

An entity can have 1..3 parents. This template uses the defined parent type on MdibEntity.

Example

A chain with 2 parents: Find numeric metrics where the parents (Channel and Vmd) have certain handles over a DFL::Mdib::Mdib.

auto r = mdib.range<NumericMetric>(
mdib.filters().chain<NumericMetric>(
[&](const DFL::NotNull<NumericMetricSharedPtr> &n) -> bool
{
return n->handle() == ehandle;
},
[&](const DFL::NotNull<ChannelSharedPtr> &c) -> bool
{
return c->handle() == p1handle;
},
[&](const DFL::NotNull<VmdSharedPtr> &v) -> bool
{
return v->handle() == p2handle;
}
)
);

Detailed Description

template<class T_Mdib>
class DFL::Mdib::MdibEntityFilters< T_Mdib >

The MdibEntityFilters template class gives predefined filter functions for Mdib iterators and ranges.

Needs a parent MDIB (to use the parent.entity() member function).
