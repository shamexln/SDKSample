Detailed Description

Encapsulates the process of authenticating a given usergroup.

The verification of a given securitycode is done by hashing it and then comparing it with the one stored in the persistence. If they match true is returned, otherwise false. But in the false case there happens a bit more. A counter is incremented counting the amount of failed login attempts. This counter is stored persistently and only set to zero after a successful verification. In parallel a second volatile counter is incremented upon verification failures. If this counter reaches a maximum of six within two minutes calling isBlocked returns true. Verification is now blocked until the blockduration is expired. Then isBlocked returns false again and a new verification attempt can be started. Basically in the background this statemachine is implemented:
