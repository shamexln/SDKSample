Detailed Description

PublisherHelper is responsible for publishing MDIB changes in the correct order based on the MdibVersion.

PublisherHelper manages DFL::Mdib::MdibObserver and SubdividedMdibObserver instances and publishes changes in the correct MdibVersion order.

The queueStateChanges or queueDescriptorChanges function is invoked during an Mdib write operation to queue all Mdib changes to a PublisherCache. Note it must be called under the transaction lock to avoid potentially skipping an MdibVersion changed on a low priority thread. The queueing function will block if no space is available in the PublisherCache.

The executePendingChanges function is responsible for generating function callback(s) and publishing them based on the MdibVersion.
