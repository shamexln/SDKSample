Detailed Description

A further specifier for Types attached to specific pm:AbstractState elements.

Is used by an SDC METRIC PROVIDER and is to be attached to pm:AbstractState elements of type pm:VmdState, pm:ChannelState, pm:AbstractMetricState or pm:AlertConditionState.

The pm:Type of an element may be specified further for example through user interaction. The SubType expressed in the pm:AbstractState can be be used to refine the pm:Type that is specified in the pm:AbstractDescriptor, similar to pm:AbstractMetricDescriptor/pm:BodySite and pm:AbstractMetricState/pm:BodySite.

Compared to an update of the pm:AbstractDescriptor, the SubType keeps the original pm:Type in the pm:AbstractDescriptor and signifies that the SubType is a more specific pm:Type.

An SDC METRIC PROVIDER shall not provide more than one SubType extension element at a time in a pm:AbstractState.

Example usage in code:

// Device side usage: create the SubType instances and add
// them after conversion with the free function
// DFL::Mdib::Ext::ParticipantModelExtensions::toExtension to any extendable
// MDIB class
auto codedValue = DFL::Mdib::CodedValueFactory::create11073cfCode("54321");
std::uint64_t const instanceNum = 44;
const SubType type{ codedValue, instanceNum };
DFL::Mdib::ChannelState const channel{
DFL::Mdib::ComponentActivation::On,
boost::none,
boost::none,
boost::none,
boost::none,
boost::none,
// just add the extension in the ctor of an MDIB class
{DFL::Mdib::Ext::ParticipantModelExtensions::toExtension(type)} };
// Client side usage: use the free function
// DFL::Mdib::Ext::ParticipantModelExtensions::getAll<SubType>()
// to retrieve the SubType from any state or other
// extendable element from the MDIB.
const auto clientSubTypes = ParticipantModelExtensions::getAll<SubType>(channel);
EXPECT_THAT(clientSubTypes, SizeIs(1u));
const auto& clientSubType = clientSubTypes[0];
EXPECT_THAT(clientSubType.type(), Eq(codedValue));
EXPECT_THAT(clientSubType.instanceNumber(), Eq(instanceNum));
