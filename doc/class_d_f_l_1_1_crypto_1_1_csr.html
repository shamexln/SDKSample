Detailed Description

Represents a certificate signing request.

A CSR is a message send to a certificate authority. It answers with a certificate which then can be used to establish an SSL connection.

DFL::Crypto::Csr::Csr ( const PrivateKey &  privateKey,
const DistinguishedName &  dn,
const Oids &  roles = Oids{} 
)

Creates a CSR ready to be send to a certificate authority.

  • Version is set to X509_V1
  • dn is used as subject name
  • roles are written in the extended key usage field
  • CSR is signed with privateKey
Exceptions
std::runtime_errorin case any of the setter fail.
const PrivateKey keyPair{HardCodedPrivateKey};
const Csr csr{keyPair, HardCodedSubjectDn, HardCodedCsrRoles};
EXPECT_TRUE(csr.csr());
EXPECT_EQ(HardCodedSubjectDn, csr.subject());
EXPECT_EQ(0L, csr.version());
EXPECT_EQ(HardCodedCsrRoles, csr.extendedKeyUsageOids());
EXPECT_EQ(PublicKey{HardCodedPublicKey}, PublicKey{csr});
EXPECT_EQ(HardCodedCsr, ::DFL::Crypto::Convert::toPem(csr));

long DFL::Crypto::Csr::version ( ) const

Returns the version number.

Note
this is defined by standards (X.509 et al) to be one less than the certificate version. So a version 3 certificate will return 2 and a version 1 certificate will return 0.
