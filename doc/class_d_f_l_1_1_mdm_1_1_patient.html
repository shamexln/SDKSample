Identifier

Contains different patient identifier such as patient account number, visit number, encouter number, medical record number (MRN).

Different systems and organizations use different identification schemes. To support the divergence, the patient model (and the MDIB) have a list of identifier. Thus it is possible to add multiple different identifier to a patient.

Two Patient instances reference the same individual in case both share at least one Mdib::InstanceIdentifier.

The identifier are stored in a sequence, where the first entry has the highest priority. This is important for interfaces where only a limited amount of identifier might be presented.

Detailed Description

Dräger model for a patient (based on MDIB patient model).

In contrast to the MDIB structures for a patient, this model may be changed depending on Dräger's needs. It combines patient information from the MDIB stored in different related structures (e.g., MDIB state and MDIB descriptor).

For better interchangeability it uses the values types and enum from the MDIB model (e.g., Mdib::InstanceIdentifier and Mdib::ContextAssociation)
