Detailed Description

A field based time point with a resolution of one day.

This is the same as a system clock time point except that the resolution of this type is days. A date can either be constructed from a year/month/day triple or from a system clock time point. This field based type is implicit convertible to the serial based type SystemDays. Any arbitrary dates can be created even some which do not exist. Use YearMonthDay::ok to check if the date is valid.

{
const YearMonthDay ymd(Year(2010), aug, Day(21));
ASSERT_TRUE(ymd.ok());
ASSERT_EQ(Year(2010), ymd.year());
ASSERT_EQ(aug , ymd.month());
ASSERT_EQ(Day(21) , ymd.day());
}
{
const YearMonthDay ymd(Year(2010), Month(13), Day(21));
ASSERT_FALSE(ymd.ok());
}
{
const YearMonthDay ymd = Year(2010)/aug/21;
ASSERT_TRUE(ymd.ok());
ASSERT_EQ(Year(2010), ymd.year());
ASSERT_EQ(aug , ymd.month());
ASSERT_EQ(Day(21) , ymd.day());
}
{
const YearMonthDay ymd(Year(2010), Month(13), Day(21));
ASSERT_FALSE(ymd.ok());
}
