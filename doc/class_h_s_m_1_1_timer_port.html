Detailed Description

template<class T = HSM::TimerInterface>
class HSM::TimerPort< T >

Input port for delivering timer events to the owning state machine and ( if implemented by TimerImpl ) on demand periodical tick events for periodical threads.

If instantiated with default parameter (HSM::TimerInterface) a timer will deliver HSM_TIMEOUT and a periodical tick will deliver HSM_TICK to the owning state machine. If instantiated with a user supplied interface definition, multiple timers may be started delivering events with different user supplied signals.

When instantiated on a periodical task, this method starts sending events every n-th run of the task.

This method only works on periodical threads! Contrary to startTimerPeriodically the period is strictly coupled to the period of the underlying thread. Delivers the event HSM_TICK unless a user supplied interface definition is used to instantiate the timer.
