Detailed Description

DEPRECATED - Define enums in a typesafe way.

Deprecated:
Use EnumClass.h instead.

Motivation

The classical enums are implicit convertible to int. This is a source of many bugs:

struct Switch
{
typedef enum {On, Off} Type;
};
Switch::Type s = Switch::On;
if(s) { cout << "On"; }
else { cout << "Off"; }
DFL::Mdib::AlertActivation::On
@ On
The element is operating.
DFL::Mdib::AlertActivation::Off
@ Off
The element is not operating.

Of course this example prints "Off" to the console.

struct State
{
typedef enum {Off, On} Type;
};
State::Type t(State::On);
switch(t)
{
case State::On: cout << "On"; break;
case State::Off: cout << "Off"; break;
default: break;
}
S31::Sdc::Impl::WorkplaceDeviceProviderImplHelper::State
State
States of the WorkplaceDeviceProviderImpl.

And it is clear that this example prints "Off" to the console, too.

There are many more scenarios where the implicit conversion of enums to int is at least confusing. The solution is to wrap an enum in a class. This way the implicit conversion is no longer available.

API

Copyable

TEST(TypesafeEnum, testCopyable)
{
Switch::Type const s1(Switch::On);
ASSERT_TRUE(s1 == Switch::On);

Switch::Type const s3(s1);
ASSERT_TRUE(s3 == Switch::On);
}

value

const Switch::Type::Native on = Switch::On.value();

ASSERT_TRUE(on != off);

To make interfacing with non typesafe components possible, a typesafe enum value can be converted to a native value.

Warning
The only guarantee this function makes is that each value has a unique number different to the other enum-values. There is especially no guarantee that the first enum-value starts with 0 nor that the others have any order.

fromValue

const Switch::Type::Native on = Switch::On.value();
const Switch::Type typesafeOn = Switch::Type::fromValue(on);
ASSERT_TRUE(typesafeOn == Switch::On);
const Switch::Type::Native invalid =
(std::numeric_limits<Switch::Type::Native>::max)();
// NOLINTNEXTLINE(cppcoreguidelines-avoid-goto)
ASSERT_THROW(static_cast<void>(Switch::Type::fromValue(invalid)),
std::runtime_error);

A typesafe enum value can also be created from a native value. If the native value does not fit to a typesafe enum value an exception is thrown.

Native value of a typesafe enum value is accessible at compiletime. This allows the usage of typesafe enum values e.g. in switch-cases and as template parameters.

Overloads

TEST(TypesafeEnum, testEachEnumValHasDifferentType)
{
struct Valve
{
bool m_on;
void turn(Switch::OnType)
{
m_on = true;
}
void turn(Switch::OffType)
{
m_on = false;
}
} valve{};
valve.turn(Switch::On);
ASSERT_TRUE(valve.m_on);
valve.turn(Switch::Off);
ASSERT_FALSE(valve.m_on);
}

Since each enum value exposes its own type one can use function overloading instead of if-else if-else implementations.
