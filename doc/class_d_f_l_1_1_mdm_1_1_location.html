Instance Identifier

Different systems and organizations use different identification schemes. To support the divergence, the location model (and the MDIB) have a list of identifier. Thus it is possible to add multiple different identifier to a location.

Two Location instances reference the same place (real-world location) in case both instances share at least one Mdib::InstanceIdentifier.

Association

Association information about the location to the corresponding device.

It represents the location association to the medical device (or the MDIB thereof). A medical device might have multiple locations. E.g., it is possible to have

  • multiple disassociated locations representing previous locations,
  • one associated location representing the current location, and
  • multiple pre-associated locations for planed future location.

Detailed Description

Dräger model of a location.

This class models the location for Dräger. It should contain data members required for communication (SDC, Medibus) as well as user interaction (for UI dialogs).
