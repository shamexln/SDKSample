Detailed Description

Encoder class to convert data into a 2 dimensional QR Code.


Example usage:

QrCode::Encoder myCode;
if (myCode.encodeData("http://www.draeger.com",22))
{
for (unsigned int x = 0; x < myCode.getWidth(); ++x)
{
for (unsigned int y = 0; y < myCode.getWidth(); ++y)
{
myDrawContext.DrawModule( xpos, ypos, myCode.isDark() ? BLACK_SQUARE : WHITE_SQUARE);
}
}
}

The class Encoder takes a given input and converts it into a two dimensional matrix which can be used to draw a QR code on the screen by a GUI application.

Restrictions:
QR specification allows 4 kinds of data to be encoded: Numeric, alphanumeric (only capital letters and some special signs), 8-bit data mode, and kanji (13 bit). The current implementation only supports 8-bit data mode.
There are 4 levels of error correction defined, the current implementation uses LEVEL_M.
This limits the maximum length of the input to 2953 bytes at lowest error correction level, 1273 bytes at highest level.
Infrastructure QR codes (using more than one picture) and Micro QR codes are not supported.
The Quiet Zone (the white margin) is fix to 4 modules.
The smallest possible QR-Code Version will be used, it depends on the size of data.
