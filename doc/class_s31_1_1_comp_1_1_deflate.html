Compressed data in gzip file format.

The bytes represent a complete valid gzip file. The gzip header has minimal information (e.g., no file name information). The gzip footer contains the CRC-32 and uncompressed size.

◆ operator+()

Deflate S31::Comp::Deflate::operator+ ( const Deflate &  rhs) const

Concatenates two separately compressed chunks into one.

Typically, all data is compressed in one go. In some cases some chunks are already compressed and need to be concatenated. The + operator concatenates two compressed deflate streams into one. The resulting deflate stream looks like it was compressed in one go, except that a flush was called in between. When decompressing it, this is not visible anymore.
