Detailed Description

A wrapper around language tags as in RFC 5646 Tags for Identifying Languages.

This is a simplified representation of language tags that provides:

  • storage of the tag (get/set)
  • get the language represented in the tag (i.e. a ISO 639 code)

Currently there is no specific support for other language tag features like 'regions' or 'scripts'. This class is just a storage, no validity checks are done on the language tag. But the class can check whether the tag is of the form ll-CC (language-country).

Examples of language tags (more in RFC 5646 - Appendix A):

  • de: German (simple Language subtag)
  • zh-Hant: Chinese written using the Traditional Chinese script (Language subtag plus Script subtag)
  • en-US: English as used in the United States (Language-Region)
  • zh-cmn-Hans-CN: (Chinese, Mandarin, Simplified script, as used in China)
  • sr-Latn-RS
  • es-419
  • invalid tags example: de-419-DE (no check is done, here allowed)

◆ is2LetterLangAnd2LetterCountry()

bool DFL::Locale::LanguageTag::is2LetterLangAnd2LetterCountry ( ) const

Checks whether the tag has the form 'll-CC' (<language>-<country>).

The language part is checked against the 2-letter codes from ISO 639-1. The country part is checked against the ALPHA-2 codes from ISO 3166-1. Other language tag parts art not allowed.

e.g.:

  • allowed: 'en-US'
  • not allowed: 'en', 'en-409', 'en-us-x-somethingextra'
Returns
true if the language tag is of the desired form.

◆ language()

std::string DFL::Locale::LanguageTag::language ( ) const

Returns the language contained in the language tag.

Results consists of a ISO 639 code (2 or 3 letters) and optional extlang parts. This is detected according the grammar described in RFC 5646. No checks is done whether the language is a valid ISO 639 code or not.

e.g.:

  • de → de
  • en-GB → en
  • zh-cmn-Hans-CN → zh-cmn
Returns
the tag if valid (according grammar), empty if invalid or no language defined in the language tag.
