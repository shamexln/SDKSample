Detailed Description

Functionality for handling metrics with different units.

Used for outgoing metrics on IACS. Units in Mdib shall not change - so there are multiple entities with different units for such cases. Only that one with right unit is active.

If a (IACS) parameter has multiple units then the according Mdib has an according metric for each possible unit, by adding '-' and a number. The sequence of added numbers has no gaps. The highest number is 9.

E.g.: IACS parameter 'P', according Mdib entities 'P', 'P-1', 'P-2',... 'P' may be an Mdib entity or not.

'P' in the example is called 'base handle'. 'P-n' in the example is called 'combined handle'.

Note: In IACS no metrics are are added or removed from Mdib. The 'normal' handle contain no '-'.
