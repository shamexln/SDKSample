Detailed Description

Executes given code when enclosing scope exits.

This functionality is extremely useful in two cases:

  • cleaning up resources (RAII)
  • rollback actions (strong-exception guarantee)
RAII (Resource Acquisition Is Initialization) is a powerful idiom in c++. It allows for binding resource acquisition and release to construction and destruction of a variable that holds the resource. Example of such types are std::unique_ptr and DFL::Thread::LockGuard. But sometimes writing such types leads to very specific classes with no reuse at all. This is where this class can be used.
It also can be used to keep a rollback action which is executed in case of a failure to satisfy the strong-exception guarantee. This class is inspired by the scope-guard feature from the d-language. scope(failure) could be implemented with the help of the c++17 function std::uncaught_exceptions. See rationale for that function here.

Use DFL::Scope::ScopeExit instead.

void DFL::ScopeExit::dismiss ( )
inlinenoexcept

Dismisses execution of given code on exit of enclosing scope.

This can be used to dismiss execution of a rollback action when the enclosing scope has not been left early e.g. by a thrown exception. This make code conform to the strong-exception guarantee.

std::string callCapture{};
{
ScopeExit onScopeExit{[&]() { callCapture += "2"; }};
// ...
// do some work which might throw
// ..
callCapture += "1";
// work has been applied successfully
// cleanup no longer necessarry
// so we call dismiss right before
// current scope is left
onScopeExit.dismiss();
}
EXPECT_EQ("1", callCapture);
