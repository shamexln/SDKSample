Detailed Description

Defines a state machine interface to orchestrate the connection process for remote devices.

The client state machine looks like depicted in the following picture.

The state machine starts in an Offline state. If a connection is triggered, a multi-step process is initiated that ends up in an Online state. Being in the Online state, at any time you can either disconnect or throw an error to stop the state machine.

Attention
The state machine is thread-safe. Only after a new state is reached the next event is accepted. There is no queuing implemented at the moment!
